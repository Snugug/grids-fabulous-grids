Grids in Graphic Design

  • Grids entered graphic design lexicon in Switzerland during WWI

  • Asymmetry became a guiding principle to feel more natural

    • Introduction of left-aligned text
    • Asymmetry, like the rule of thirds in photography, makes content feel more dynamic and engaging

  • Grids became entrenched in Graphic Design, even if they aren't always straight-forward

    • Famous posters by Josef Muller Brockmann (left), Richard P. Lohse, Armin Hofmann and Alberto Longhi (right)
    • Each conforms to a grid (each has a unique asymmetric grid)

A number page layout terms emerged from this time

Column
Area where content (often text) exists
Gutter
Space between columns
Rail
A thinner column (typically about half the width of the other columns) at the far left or far right of a page
Single column grid
One column of content, like in a typical book
Multi-column (vertical) grid
More that one column of content, like in a typical newspaper
Modular grid
A grid that involves about as many horizontal divisions as it does vertical ones, making for an especially flexible model of text and image arrangement
Bonus: Span
An area of content across more than one column

History of the Grid

Web Grids

Grids on the web have traditionally been hard for almost the precise opposite reasons print grids are easy

  • Many and varied display sizes
  • Variable content lengths, sizes, and types
  • Because of these problems, we've traditionally had to abandon vertical grids on the web * We also tried, for a while, to remove some of this variance by fixing the sizes of our sites

In September of 2006, Cameron Moll discussed creating the optimal website width for a 1024px resolution screen.

In it, he discusses the need to move from 800x600 resolution to 1024x768

In it, he proposes using 960px as the full-width of a site as "it‚Äôs slightly smaller than full width, and it‚Äôs divisible by 3, 4, 5, 6, 8, 10, 12, 15, and 16"

  • In the article, he doesn't actually recommend this as a fixed width site, but rather optimizing for that resolution
  • A follow-up article that December, Gridding the 960, became the inspiration for the most influential web grid to date, the 960 Grid System

960 Grid System

  • First release was March of 2008

  • 960gs comes with 2 column layouts: 12 and 16 column, and does not have a vertical column, with 12 being the most widely used

  • The premise was originally for rapid prototyping, although it does say that it would work equally well for production

  • Markup relied heavily on nesting of divs, with creator Nathan Smith taking a strong stance that semantics were not in class names or div nesting, but in the actual tags used and content in the tags

    • He's right there, but it also led to lots of messy, hard-to-follow markup

  • It was a big hit because it included not only CSS, but templates in all of the major graphic editors at the time

  • ‚Ä¶Then, on May 25, 2010, Ethan Marcotte released his A List Apart article "Responsive Web Design" and static grids had to be rethought

  • In August and September of 2011, Twitter Bootstrap and Zurb Foundation had their first respective releases
  • Both came with a 12 column 960-based grid that replicated its functionality (both in markup and class structure).
  • All of these grids were based on float layout models. In 2012, though, we got a new layout model to work with; flexbox.
    • This is the "tweener" syntax, the first implementation of the final syntax, unprefixed, would show up in 2013

Grids, Flexbox, and Floats. Oh My!

When it comes to layout performance, Flexbox is faster than Float for the render engine to lay out a page

With browser-defaults, CSS Grid is faster than Flexbox to lay out a page

  • Float

    • No direct layout control
    • Pushes items either towards the left or the right for indirect layout control
    • No built-in concept of spacing

  • Flexbox

    • Lays out items in one dimension
    • Size and position based on content size
    • Cannot control positioning in more that one direction
    • No built-in concept of spacing

  • Grids

    • Lays out items in two dimensions
    • Set constraints for size and position
    • Can directly control positioning in each direction
    • Has a built-in concept of spacing

Creating Grids

Columns and Rows

  • Content areas running across the view are columns
  • Content areas running down the view are rows
  • Grids must have at least one column and one row

  • When choosing the content areas, there is a balance that needs to be made between flexibility and constraints

    • A symmetric grid, where each content area is the same size offers greater flexibility but fewer constraints
    • An asymmetric grid, where content areas are of different sizes, offer greater constraints with less flexibility

  • Constraints are good!

    • Simpler grids with less possible variations lead to cleaner hierarchies for content
    • Too many options leads to over-thinking layouts! The goal of a grid system is to minimize the thought processes to keep your mind free to find more creative opportunities for expression
    • Using asymmetrical, uneven content areas, much like the rule of thirds, allows for more dynamic, engaging layouts that fir the needs of your imagery, text, or whatever else
    • Less options also means less to adjust for responsive design, allowing hierarchy and compositions to scale more fluidly

Breaking out of the 12 column rut

Gutters

  • Called gap in CSS Grids
  • Space between each content area
  • Gutters are all always the same size, although columns and rows can have different gutters

Span

  • Spanning is how you attach a content to a column or row
  • Spanning more than one column or row also spans the gutter separating those columns or rows

The most flexible grid ‚Äì giving us the most possibilities for layout ‚Äì would be a column for every pixel right? How useless would that be? Flexibility is not solely what we seek in a grid system. We want a harmonious and meaningful utilization of space to house our content. This includes proportions between elements, as well as the proportions of negative space. Columns are a crutch. Less columns means less options for layout, reinforcing structure, creating a stronger content hierarchy, and therefore improving the overall clarity of communication.

CSS Grids

Grid Markup

<Grid>
  <Grid-Item></Grid-Item>
  <Grid-Item></Grid-Item>
  <Grid-Item></Grid-Item>
</Grid>
  • CSS Grid markup is pretty flat. You have a parent element that has the grid definition on it, and that grid definition applies to its direct children and only its direct children
  • The grid definition can contain columns and rows, and the items can be arranged anywhere on their parent's grid

Defining a Grid

  • grid-template-columns and grid-template-rows define the columns and rows for a grid: grid-template-columns: 200px 300px 400px
  • New units are available for defining templates! fr, ch, and auto
  • Sections of a template can be repeated with repeat: grid-template-columns: repeat(12, 1fr)
  • Templates can be based on constraints. min, max, minax: grid-template-rows: minmax(2em, auto) auto;
  • Gaps can be set either for the whole grid (grid-gap) or for columns and rows individually (grid-column-gap and grid-row-gap)

  • Units

    • fr - Fractional unit, 1 part of the available space
    • ch - Width, or more precisely the advance measure, of the "0" (zero, Unicode U+0030) in the element's font
    • auto - Maximum space needed for the content, or largest minimum size of items the items on its track (row/column)

  • Constraints

    • min - Smallest the item can be
    • max - Largest the item can be
    • minmax - Constraint of small to large an item can be

Spanning a Grid

  • grid-column and grid-row allow a child of a grid to span its parent's grid
  • Spanning can be start / end, start / span, or a single value span that will auto-place items spanning that number of columns or rows
    • grid-column: 2 / span 3 will span 3 columns starting at the 2nd column
    • grid-row: span 2 will span 2 rows starting at the last row available

Areas

.container {
  display: grid;
  grid-template-columns: 1fr minmax(1fr, 80ch) 1fr;
  grid-template-rows: 2em auto auto;
  grid-template-areas:
    'header header header'
    'left-side main right-side'
    'footer footer footer';
}

.header {
  grid-columns: header;
}

.main {
  grid-columns: main;
}
  • Grid Areas allow for easier use and spanning of grid templates
